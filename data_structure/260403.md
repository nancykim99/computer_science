## 1. 복잡도

### 시간 복잡도 & Big-O

- **Big-O** = 최악의 경우 기준으로 점근적 복잡도를 표현
- 상수·낮은 차수 항 무시, 가장 지배적인 항만 표기
- 순서: O(1) < O(log n) < O(n) < O(n log n) < O(n²)

| 복잡도 | 예시 |
|---|---|
| O(1) | 배열 인덱스 접근, 해시테이블 조회 |
| O(log n) | 이진 탐색, 균형 BST |
| O(n) | 배열 순차 탐색, 반복문 1개 |
| O(n log n) | 병합 정렬, 퀵 정렬 평균 |
| O(n²) | 버블 정렬, 이중 반복문 |

### 공간 복잡도

- 알고리즘이 사용하는 **메모리 양**을 나타냄
- 입력 데이터 크기 + 추가로 사용하는 메모리 고려

## 2. 선형 자료구조

### 배열 vs 연결 리스트

| | 배열 | 연결 리스트 |
|---|---|---|
| 메모리 | 연속 | 비연속 (포인터 연결) |
| 인덱스 접근 | O(1) | O(n) |
| 삽입/삭제 | O(n) | O(1) (위치 알 때) |
| 캐시 효율 | 좋음 | 나쁨 |

> 배열 인덱스 접근이 O(1)인 이유: `주소 = 시작주소 + 인덱스 × 원소크기`

### 동적 배열 (벡터)

- `push_back` = 분할 상환 O(1)
- 꽉 차면 **2배 확장 + 복사** O(n)
- n번 수행 시 총 복사 횟수 ≈ 2n → 1번 평균 O(1)

### 스택 vs 큐

| | 스택 | 큐 |
|---|---|---|
| 방식 | LIFO (후입선출) | FIFO (선입선출) |
| 활용 | 함수 콜 스택, 괄호 검사, 브라우저 뒤로 가기, DFS | 프로세스 스케줄링, 프린터 스풀러, BFS |

#### 스택 두 개로 큐 구현

```
inbox(삽입용) + outbox(추출용)
enqueue → inbox에 push
dequeue → outbox 비면 inbox 전체 이동 후 pop
→ amortized O(1)
```

## 3. 비선형 자료구조

### 그래프 vs 트리

| | 트리 | 그래프 |
|---|---|---|
| 사이클 | 없음 | 있을 수 있음 |
| 루트 | 1개 | 없음 |
| 간선 수 | n-1 (n개 노드) | 제한 없음 |
| 정의 | 비순환 연결 무방향 그래프의 특수 형태 | 제약 없음 |

### 인접 행렬 vs 인접 리스트

| | 인접 행렬 | 인접 리스트 |
|---|---|---|
| 공간 | O(V²) | O(V+E) |
| 연결 확인 | O(1) | O(V) |
| 적합한 경우 | 밀집 그래프 | 희소 그래프 (실무 주로 사용) |

### BFS vs DFS

| | BFS | DFS |
|---|---|---|
| 방향 | 너비 우선 | 깊이 우선 |
| 자료구조 | 큐 | 스택/재귀 |
| 탐색 방식 | 레벨 단위 | 한 방향으로 끝까지 |
| 활용 | 최단 경로 (가중치 없을 때) | 경로 존재 여부, 사이클 탐지 |

### BST (이진 탐색 트리)

- 규칙: **왼쪽 < 현재 노드 < 오른쪽**
- 평균: O(log n) / 최악: O(n) → 한쪽으로 쏠릴 때 (1,2,3,4,5 순서 삽입)

### AVL vs Red-Black 트리

| | AVL 트리 | Red-Black 트리 |
|---|---|---|
| 균형 기준 | 좌/우 높이 차 ≤ 1 (엄격) | 색상 규칙 (덜 엄격) |
| 유리한 경우 | 탐색 | 삽입/삭제 |
| 예시 | — | Java TreeMap, C++ std::map |

### 트리 순회

| 순회 | 순서 | 활용 |
|---|---|---|
| 전위 (Pre) | 루트 → 왼 → 오 | 트리 복사, 직렬화 |
| 중위 (In) | 왼 → 루트 → 오 | BST 오름차순 정렬 |
| 후위 (Post) | 왼 → 오 → 루트 | 트리 삭제, 수식 계산 |

### 힙 (Heap)

- **완전 이진 트리** 기반
- 최대 힙: 부모 ≥ 자식 → 루트 = 최댓값
- 최소 힙: 부모 ≤ 자식 → 루트 = 최솟값

| 연산 | 복잡도 | 방법 |
|---|---|---|
| 삽입 | O(log n) | 마지막 추가 후 heapify-up |
| 삭제 | O(log n) | 루트 제거 후 heapify-down |
| 조회 | O(1) | 루트만 확인 |

> 배열 인덱스: 부모=i/2, 왼자식=2i, 오른자식=2i+1

### 우선순위 큐 vs 힙

- **우선순위 큐** = ADT(개념): 우선순위 높은 원소 먼저 추출
- **힙** = 구현체: 우선순위 큐를 가장 효율적으로 구현한 자료구조

### 해시 테이블

```
index = hash(key) % table_size
평균 O(1) / 최악 O(n)
```

#### 충돌 해결 방법

| 방법 | 설명 | 예시 |
|---|---|---|
| 체이닝 | 같은 버킷에 연결 리스트로 연결 | Java HashMap (8개 초과 시 Red-Black 트리 전환) |
| 개방 주소법 | 빈 슬롯 탐색 (선형/이차/이중 해싱) | — |

- **로드 팩터** = 저장 원소 수 / 테이블 크기
- 0.75 초과 시 **리해싱** (2배 확장 + 재삽입)

#### 좋은 해시 함수 조건

1. 결정론적: 같은 key → 항상 같은 값
2. 균일 분포: 골고루 퍼짐
3. 계산 효율성: 빠른 계산
4. 눈사태 효과: 작은 변화 → 큰 hash 변화

### 정렬 알고리즘 비교

| 알고리즘 | 평균 | 최악 | 안정 정렬 | 메모리 |
|---|---|---|---|---|
| 퀵 정렬 | O(n log n) | O(n²) | ❌ | O(log n) |
| 병합 정렬 | O(n log n) | O(n log n) | ✅ | O(n) |
| 힙 정렬 | O(n log n) | O(n log n) | ❌ | O(1) |

> 퀵 정렬 최악 조건: 이미 정렬된 배열 + 첫 번째 원소를 피벗으로 선택할 때
> 해결책: 랜덤 피벗, Median of Three

### 이진 탐색

- **조건**: 반드시 정렬된 배열
- 시간복잡도: O(log n)
- 방법: 중간값과 비교 → 작으면 왼쪽, 크면 오른쪽 반씩 좁혀감

### 최단 경로 알고리즘

| 알고리즘 | 조건 | 복잡도 |
|---|---|---|
| BFS | 가중치 없는 그래프 | O(V+E) |
| 다익스트라 | 음수 가중치 없음 | O((V+E)log V) |
| 벨만-포드 | 음수 가중치 허용 | O(VE) |
| 플로이드-워셜 | 모든 쌍 최단 경로 | O(V³) |

## 💬 면접 예상 질문

**Q. 배열과 연결 리스트의 차이는?**
> 배열은 연속 메모리로 인덱스 접근 O(1)이지만 삽입/삭제 O(n). 연결 리스트는 비연속 메모리로 접근 O(n)이지만 삽입/삭제 O(1).

**Q. 스택과 큐의 차이는?**
> 스택은 LIFO(후입선출), 큐는 FIFO(선입선출). 스택은 DFS·함수 호출에, 큐는 BFS·스케줄링에 활용.

**Q. 힙과 우선순위 큐의 차이는?**
> 우선순위 큐는 우선순위 높은 원소를 먼저 꺼내는 ADT(개념)이고, 힙은 그것을 구현하는 자료구조.

**Q. 해시 충돌 해결 방법은?**
> 체이닝(연결 리스트로 연결)과 개방 주소법(빈 슬롯 탐색)이 있음. Java HashMap은 체이닝 방식.

**Q. BST의 최악 시간복잡도가 O(n)인 이유는?**
> 정렬된 순서로 삽입 시 한쪽으로만 쏠려 선형 구조가 됨. AVL/Red-Black 트리로 O(log n) 보장.
